$ cd Reveal-Remote

El Problema

El orador atado al escritorio. La audiencia pierde conexión.

🔗
Sin movilidad
Cada slide requiere volver al teclado. Rompe el ritmo de la charla.
📲
Setup complejo
Apps, registros, configurar redes. Demasiado antes de empezar.
📡
Dependencia de red
WiFi caído o red corporativa restrictiva = la charla se complica.
⚙️
Sobreingeniería
Soluciones pesadas para un problema simple: pasar diapositivas.

Flujo de Conexión

Del QR al control en 2 segundos

1 node server.js — Express sirve /remote y la API
2 La PC abre WebSocket y se registra: { type: "display" }
3 El QR se genera y se muestra en pantalla
4 El celular escanea, abre el control y se conecta al WebSocket
5 Listo — cada toque viaja: controller → relay → display → Reveal.js

Protocolo

Mensajes JSON sobre WebSocket. Simple y extensible.

📤 Controller → Servidor
Registro del control remoto
{ type: "controller" }
Navegación (← → ↑ ↓)
{ action: "left" }
{ action: "right" }
{ action: "up" }
{ action: "down" }
📥 Servidor → Display
Reenvío de comandos
{ action: "right" }
Control conectado
{ type: "controller-connected" }
Estado de la diapositiva
{ type: "slideInfo", index: 6, total: 11, fragment: 2, totalFragments: 2 }

Servidor

El núcleo del sistema en menos de 120 líneas de Node.js

1 Escuchar conexiones
El servidor abre un WebSocket en el mismo puerto que Express. Espera conexiones de display (PC) y controller (celular).
2 Clasificar según el tipo
Cada cliente envía un mensaje de registro:
{ type: "display" } → se guarda como display { type: "controller" } → se agrega a la lista
3 Reenviar al display
Cuando el controller envía un { action: "right" }, el servidor lo reenvía al display. El display ejecuta Reveal.next().

⌨️
Keyboard support

Reconexión automática
Backoff exponencial (2s → 15s). Indicador visual de estado.
